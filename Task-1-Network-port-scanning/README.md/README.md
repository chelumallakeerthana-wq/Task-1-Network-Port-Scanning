Task 1: Network Port Scanning and Security Assessment


Project Name
Network Port Scanning and Security Assessment using Kali Linux, Nmap, and Wireshark**



 Objective

The objective of this project is to identify active hosts and discover open TCP ports within an authorized local network using Nmap. Additionally, Wireshark is used to capture and analyze network traffic generated during the scanning process to better understand TCP communication and network exposure.



Tools Used

* Kali Linux
* Nmap 7.99
* Wireshark
* VMware Workstation
* Windows 11 (Host Operating System)



 Methodology

1. Verified the installation of Nmap.
2. Identified the local network configuration.
3. Performed a TCP SYN scan using Nmap.
4. Identified active hosts and analyzed scanned ports.
5. Captured network traffic using Wireshark during the Nmap scan.
6. Examined TCP SYN and TCP RST/ACK packets.
7. Saved scan results in TXT and XML formats.
8. Documented findings and provided security recommendations.



 Commands Used

```bash
nmap --version

ip addr

sudo nmap -sS <masked-network>/24

sudo nmap -sS <masked-network>/24 -oN Scan_Result.txt

sudo nmap -sS <masked-network>/24 -oX Scan_Result.xml




 Results

* Successfully performed a TCP SYN scan on an authorized local network.
* Host discovery and port scanning were completed successfully.
* No open TCP ports were detected during the scan.
* Wireshark successfully captured the network traffic generated by the Nmap scan.
* TCP SYN packets and TCP RST/ACK responses were observed during packet analysis.



Security Risks

* Open ports may expose network services to unauthorized users.
* Firewalls help reduce network exposure by filtering unwanted traffic.
* Regular network assessments help identify unnecessary exposed services.
* Proper system updates and secure configurations reduce security risks.



 Recommendations

* Keep firewall protection enabled.
* Disable unused network services.
* Regularly perform security assessments using Nmap.
* Monitor suspicious network activity using Wireshark.
* Keep operating systems and applications updated with the latest security patches.

Conclusion

This project demonstrated the use of Kali Linux, Nmap, and Wireshark to perform network reconnaissance and analyze TCP communication within an authorized environment. The assessment identified no exposed TCP ports during the scan, while Wireshark successfully captured the packet exchange involved in the TCP SYN scanning process. The project highlights the importance of regular network assessments and proper security configurations to minimize the network attack surface.
